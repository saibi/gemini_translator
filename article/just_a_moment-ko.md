# Claude Code(와 Cowork)로 AI 마케팅 팀을 구축한 방법

**Snow W. Lee** 저 | 2026년 3월 | [snow.runbear.io](https://snow.runbear.io)

---

나는 코드를 한 줄도 직접 쓰지 않고 AI 마케팅 팀을 만들었다. Claude가 전부 다 썼다.

이 글은 Claude Code와 에이전트 팀(agent teams) 기능을 활용해 실제로 동작하는 자율 마케팅 시스템을 어떻게 구축했는지에 대한 이야기다. 이 시스템은 내가 자는 동안에도 블로그 포스트를 발행하고, 소셜 미디어 계정을 관리하고, SEO 성과를 분석한다.

---

## 첫 번째 세션

모든 것은 단 하나의 Claude Code 세션에서 시작되었다.

나는 Claude에게 우리 제품에 대해 설명했다. 제품이 무엇을 하는지, 누구를 위한 것인지, 어떤 경쟁사들이 있는지, 그리고 우리의 가치 제안(value proposition)이 무엇인지. 그런 다음 Ahrefs MCP 서버를 연결해서 Claude가 현재 키워드 랭킹을 직접 가져오고 검색 엔진에서의 우리 위치를 확인할 수 있게 했다.

그리고 go-to-market 전략을 만들어달라고 요청했다.

Claude는 90일 콘텐츠 플랜을 내놓았다. SEO 기둥 키워드(SEO pillar)에 매핑된 블로그 포스트 주제들, 소셜 미디어 배포 케이던스(cadence), 플랫폼 우선순위, 주간 마일스톤까지 갖춰진 플랜이었다.

그 순간이 내가 이것이 단순한 실험 이상이 될 수 있다는 걸 처음으로 느낀 때다.

## 핵심 인사이트: 프롬프트가 아닌 직무 기술서(Job Description)로 생각하라

처음엔 나도 다른 사람들처럼 접근했다. 프롬프트를 작성하고, 결과를 보고, 더 나은 프롬프트를 쓰는 방식. 그런데 이 사고방식으로는 확장이 안 된다.

전환점은 **프롬프트가 아닌 직무 기술서**로 생각하기 시작했을 때였다.

실제 마케팅 팀을 고용한다면 어떻게 할까? 직원 핸드북을 쓰는 대신, 마케터에게는 마케터의 업무를 정의하고, 콘텐츠 라이터에게는 그들의 역할을 문서화할 것이다. 이 아이디어를 그대로 Claude에 적용했다.

각 에이전트는 `.claude/agents/` 디렉토리 안에 단일 마크다운 파일로 존재한다. Claude Code는 이 파일들을 읽고, CMO가 작업을 위임할 때 해당 전문 서브에이전트(subagent)를 소환(spawn)한다.

## 에이전트 팀(Agent Teams) 작동 방식

이 전체 시스템은 **Claude Code의 실험적 기능인 에이전트 팀**을 기반으로 한다. 이는 옵트인(opt-in) 기능으로, 하나의 에이전트가 다른 에이전트들을 서브프로세스로 소환하고 조율할 수 있게 해준다.

구체적으로 동작하는 방식은 다음과 같다:

1. **CMO 에이전트**가 메인 에이전트 역할을 한다.
2. CMO는 Claude Code의 `Agent` 도구를 사용해 **전문 서브에이전트**를 소환한다.
3. 각 서브에이전트는 자체적인 **격리된 컨텍스트 윈도우(context window)**, 시스템 프롬프트, 그리고 도구 세트를 갖는다.
4. 전문 에이전트들은 **병렬로 실행**될 수 있다. 오늘 슬롯이 블로그 포스트 발행과 소셜 미디어 인게이지먼트 세션을 동시에 요구한다면, CMO는 콘텐츠 라이터와 소셜 미디어 마케터를 동시에 소환할 수 있다.

현재 팀 구성은 다음과 같다:

- **CMO** — 전략 수립, 에이전트 조율, 주간 계획 실행
- **Content Writer (콘텐츠 라이터)** — 블로그 포스트 작성 및 발행
- **Social Media Marketer (소셜 미디어 마케터)** — 소셜 채널 관리 및 인게이지먼트
- **SEO Analyst (SEO 분석가)** — 키워드 분석 및 성과 추적

## 주간 계획(Weekly Plan): 시스템의 핵심

시스템에서 가장 중요한 파일은 **주간 계획 문서**다.

이 파일은 요일별, 시간대별(3시간 단위 슬롯)로 각 에이전트가 정확히 무엇을 해야 하는지를 정의한다. CMO가 활성화되면, 현재 시간을 확인하고, 올바른 슬롯을 찾은 뒤 그 계획에 따라 작업한다.

모든 에이전트가 자신이 어느 시간 슬롯에 있는지, 그 슬롯에서 정확히 무엇을 해야 하는지 알고 있기 때문에, 매시간 실행되는 cron 작업에서는 크게 설명할 필요가 없다. 주간 계획 파일이 모든 것을 처리한다.

---

## 자동화: cron + Claude Code 헤드리스 모드

자동화는 아주 단순한 crontab 항목 하나로 구현된다.

```bash
# 매 시간 정각에 CMO 에이전트 실행
0 * * * * cd /path/to/marketing-project && claude --dangerously-skip-permissions \
  -p "You are the CMO. Check the current time, read the weekly plan, and execute the current time slot's tasks." \
  > /dev/null 2>&1
```

`--dangerously-skip-permissions` 플래그는 시스템을 **완전 자율(fully autonomous)** 모드로 만든다. 파일 편집, 스크립트 실행, API 호출 등 모든 작업에서 수동 승인을 요구하지 않는다. 에이전트들이 감독 없이 엔드투엔드로 실행될 수 있다.

> **주의**: `--dangerously-skip-permissions`는 강력하지만 위험할 수 있는 플래그다. 에이전트들이 시스템에 대한 광범위한 접근 권한을 가지게 되므로, 신뢰할 수 있는 코드베이스와 잘 정의된 에이전트 파일이 있을 때만 사용해야 한다.

## Slack 통합: 팀 스탠드업처럼 읽히는 채널

에이전트들은 작업을 완료할 때마다 프라이빗 Slack 채널인 **#team-ai-marketing**에 보고한다. 각 에이전트는 Slack 스크립트에서 `--username` 플래그를 사용해 자신의 이름으로 메시지를 전송한다.

그 결과는 팀 스탠드업처럼 읽히는 Slack 채널이다. 단지 내가 자는 동안 실행될 뿐이다.

```
[Content Writer] 블로그 포스트 "Claude Code로 SEO 자동화하기" 발행 완료.
  - 워드 수: 1,847
  - 타겟 키워드: "claude code seo"
  - Ghost CMS에 스케줄링됨: 오전 9시

[Social Media Marketer] LinkedIn, Twitter 게시 완료.
  - LinkedIn: 187 impressions (첫 1시간)
  - Twitter 스레드 3개 팔로업 인게이지먼트 완료

[SEO Analyst] 주간 키워드 랭킹 보고서 생성 완료.
  - 상승 키워드: 12개
  - 하락 키워드: 3개 (대응 필요)
```

## 하드웨어 설정: 침실의 Mac Mini M1

이 시스템은 침실에 놓인 **Mac Mini M1**에서 실행된다. 조용하고, 전력 소비가 적으며, 실행 중에 절전 모드로 들어가지 않는다. 항상 켜져 있는 전용 머신이다.

팀이 막히거나 세션을 직접 확인하고 싶을 때는 **Jump Desktop**을 폰이나 노트북에서 열어서 Mac Mini 화면을 몇 초 만에 볼 수 있다.

## 에이전트 메모리: 기관 지식(Institutional Knowledge)을 가진 팀

각 에이전트는 세션 간에 유지되는 **영구 메모리 디렉토리**를 가진다.

- **CMO**는 승인된 전략, 실행한 캠페인, 반응이 좋았던 메시지를 기억한다.
- **소셜 미디어 마케터**는 이미 답글을 달았던 계정을 기억해서 중복 인게이지먼트를 피한다.

이렇게 하면 시스템이 기관 지식을 가진 팀처럼 느껴진다. 에이전트들이 점점 더 똑똑해지는 것이다.

---

## 매니저로서의 역할

시스템이 실행되기 시작하면서, 내 역할은 **실행(execution)에서 관리(management)로** 바뀌었다.

이제 나는 Claude Code에서 이런 식으로 타이핑한다:

```
소셜 미디어 마케터를 업데이트해서 어떤 플랫폼에서도
절대 em dash를 사용하지 않도록 해줘.
```

그러면 Claude가 에이전트 파일을 직접 수정한다. 한 번의 지시로 에이전트의 행동이 영구적으로 바뀐다.

이것이 일반적인 프롬프트와의 근본적인 차이다. 프롬프트는 그 대화에서만 동작한다. 직무 기술서를 업데이트하면 에이전트의 모든 미래 행동에 영향을 미친다.

## Cowork와의 연결

이 시스템은 **Claude Cowork**와도 함께 활용할 수 있다. Cowork는 웹을 탐색하고, 페이지를 읽고, 폼을 작성하고, 정보를 추출할 수 있다.

마케터 입장에서 이는 다음을 의미한다:

- 도구를 전환하지 않고도 경쟁사 리서치
- 가격 비교 및 포지셔닝 분석
- 소셜 미디어 모니터링

Claude Cowork의 Skills 기능은 마크다운 파일로 AI 에이전트에게 특화된 지식과 워크플로우를 부여한다. 에이전트들이 특정 마케팅 작업을 수행할 때 올바른 프레임워크와 모범 사례를 자동으로 적용하게 된다.

## 비용

전체 시스템은 **하루 약 $20~50의 Claude API 사용료**가 든다. 실행되는 블로그 포스트와 소셜 미디어 세션 수에 따라 달라진다.

풀타임 마케팅 채용에 비하면 반올림 오차(rounding error) 수준이다.

---

## 이것이 가능하게 된 이유

이 시스템이 실제로 동작하는 이유는 크게 두 가지다:

1. **에이전트 팀(Agent Teams)**: Claude Code의 실험적 기능으로, 한 에이전트가 다른 에이전트들을 서브프로세스로 조율할 수 있다. 격리된 컨텍스트, 목적에 맞게 만들어진 시스템 프롬프트, 병렬 실행 — 이것이 단순한 단일 에이전트와의 차이를 만든다.

2. **직무 기술서로서의 마크다운**: 프롬프트 엔지니어링이 아닌 조직 설계로 접근했다. 각 에이전트는 역할, 책임, 행동 방식이 명확히 정의된 직원이다.
